1. pipenv

1.1 安装虚拟环境

可在当前目录下创建 Pipfile 、 Pipfile.lock 文件，在当前目录下新增一个虚拟环境，虚拟环境文件夹的目录名称的形式为“当前项目目录名+一串随机字符”，比如 helloflask-LW-PdfAB。

虚拟环境默认被安装到的地址:
C:\Users\ASUS\.virtualenvs\helloflask-LW-PdfAB

虚拟环境默认被安装到的地址:
~/.local/share/virtualenvs/helloflask-LW-PdfAB

1.4 删除虚拟环境

运行别人的flask程序，一定要确定好别人的环境各个包的版本 ！！！
最最最好用 pycharm + requirements.txt 来管理环境

在学习李辉大神的python web开发实战这本书的时候，运行他的示例程序，可以从他给的 https://github.com/greyli/helloflask/ 找到 requirements.txt

补充：pycharm用requirements.txt来配置自己的环境

  • 新建一个文件夹 [项目的根目录]
  • 用 pipenv 安装虚拟环境，并启动虚拟环境。
  • 访问李辉老师 github 上 flask web 书前四章的环境 requirements ：https://github.com/greyli/helloflask/blob/master/requirements/v1.txt 把里面所需模块拷到 requirements.txt 中。
  • 将 requirements.txt 保存导项目根目录下
  • 在虚拟环境下，执行 pip install -r requirements.txt。发现报错

仔细观察发现， error in jsmin setup command: use_2to3 is invalid. 罪魁祸首是这个叫 jsmin 的包。这个包正好也在 requirements.txt 所列的包里面，所以我们判断应该没错。那么就先不安装这个包，把他暂时先从 requirements.txt 中先移除，再执行 pip install -r requirements.txt 安装。然后顺利安装成功，至于这个叫 jsmin 的包，用到再说。

3. url_for 函数

使用 url_for() 函数生成的 URL 是相对 URL（即内部 URL ），即 URL 中的 path 部分，比如 "/hello" ，不包含根 URL 。

调用 url_for('index') 即可获取对应的 URL ，即 "/" 。

调用 url_for('greet'，name='Jack') 得到的 URL 为 "/greet/Jack" 。

验证器 说明

7.1 连接数据库服务器

在 Flask-SQLAlchemy 中，数据库的 URI 通过配置变量SQLALCHEMY_DATABASE_URI 设置，默认为 SQLite 内存型数据库 （sqlite：///：memory：） 。SQLite 是基于文件的 DBMS ，不需要设置数据库服务器，只需要指定数据库文件的绝对路径。

注意： SQLite 的数据库 URI 在 Linux 或 macOS 系统下的斜线数量是4个；
在 Windows 系统下的 URI 中的斜线数量为3个。内存型数据库的斜线固定为3个。
